# Healthcare SQL Portfolio

A hands-on portfolio of SQL and healthcare-data analytics projects, built on **synthetic patient data** using **Microsoft SQL Server**. The goal of this work is to develop and demonstrate the technical skills required for an entry-level **EHR / Epic Applications Analyst** role.

> ⚕️ **All data in this repository is synthetic.** Patient records are generated by [Synthea](https://github.com/synthetichealth/synthea) and contain **no real protected health information (PHI)**.

---

## About this project

I'm building toward a role as an EHR Applications Analyst in an enterprise healthcare network, with the longer-term goal of employer-sponsored Epic certification. Because Epic's reporting layer (Clarity, Caboodle) is built on SQL Server, this portfolio develops those exact skills: writing SQL against realistic clinical data, building reusable reporting views, and translating clinical/operational questions into data answers.

Each project below is framed the way an analyst actually receives work — as a plain-English question from a stakeholder — and documented end to end.

## Tech stack

- **Microsoft SQL Server 2022** (Developer Edition) + SQL Server Management Studio (SSMS)
- **Synthea** synthetic patient data (CSV and FHIR formats)
- **Power BI Desktop** (reporting & visualization)
- **Git / GitHub** (version control & documentation)
